《Java与模式》对适配器模式的描述如下：
适配器模式把一个类的接口变换成客户端所期待的另一种接口，从而使原本因接口不匹配而无法在一起工作的两个类能够在一起工作。

　　适配器模式最经典的例子就是充电器插头的描述，我们都知道世界各地的插座都不一样，比如我们从香港代购过来是苹果手机，给配的充电器都是大头的，那是因为香港的插座都是大头。这时候想给手机充电，就需要我们自己买一个转换器，把大头转换成国内通用的小头。

　　适配器模式分成类的适配器模式和对象的适配器模式两种方式。

类的适配器模式

结构

　　类适配器模式所涉及的角色有：

  • 目标(Target)角色：这就是所期待得到的接口。注意：由于这里讨论的是类适配器模式，因此目标不可以是类。

  • 源(Adapee)角色：现在需要适配的接口。

  • 适配器(Adaper)角色：适配器类是本模式的核心。适配器把源接口转换成目标接口。显然，这一角色不可以是接口，而必须是具体类。

　　　类适配器模式的类图如下：

代码实例

目标角色

package com.javapatterns.adapter;
//目标接口
public interface Target {
　　//这是源类也有的方法
　　void sampleOperation1();

　　//这是源类没有的方法
　　void sampleOperation2();
}

源角色

package com.javapatterns.adapter;
//源类
public class Adaptee {
　　//源类已经实现的方法
　　public void sampleOperation1() {
　　　　System.out.println("the method sampleOperation1 in Adaptee..");
　　}
}

适配器角色

package com.javapatterns.adapter;
//适配器类
public class Adapter extends Adaptee implements Target {

　　//因为源类没有实现这个方法所以适配器需要提供该方法
　　@Override
　　public void sampleOperation2() {
　　　　System.out.println("the method sampleOperation2 in Adapter..");
　　}
}

实现效果

　　从上面的代码和类图中可以看出来，Adaptee没有方法sampleOperation2()，而客户端又希望使用这个方法。为了在保证sampleOperation1()方法不丢失的情况下，我们加入一个适配器类Adapter，它在没有改变Adaptee的sampleOperation1()方法的情况下，加入了客户端需要的sampleOperation2()方法。

对象的适配器模式

结构

　　对象适配器模式所涉及的角色有：

  • 目标(Target)角色：这就是所期待得到的接口，目标可以是具体的或抽象的类。

  • 源(Adapee)角色：现在需要适配的接口。

  • 适配器(Adaper)角色：适配器类是本模式的核心。适配器把源接口转换成目标接口。显然，这一角色必须是具体类。

　　　类适配器模式的类图如下：

代码实例

目标角色

package com.javapatterns.adapter2;
//目标接口
public interface Target {
　　//这是源类也有的方法
　　void sampleOperation1();

　　//这是源类没有的方法
　　void sampleOperation2();
}

源角色

package com.javapatterns.adapter2;
//源类
public class Adaptee {
　　//源类已经实现的方法
　　public void sampleOperation1() {
　　　　System.out.println("the method sampleOperation1 in Adaptee..");
　　}
}

适配器角色

package com.javapatterns.adapter2;
//适配器类
public class Adapter implements Target {
　　//创建一个源类的实例对象
　　private Adaptee adaptee;

　　public Adapter(Adaptee adaptee) {
　　　　this.adaptee = adaptee;
　　}

　　//调用源类自己的sampleOperation1方法
　　@Override
　　public void sampleOperation1() {
　　　　this.adaptee.sampleOperation1();
　　}

　　//实现一个sampleOperation2方法
　　@Override
　　public void sampleOperation2() {
　　　　System.out.println("the method sampleOperation2 in Adapter..");
　　}
}

实现效果

　　类的适配器模式有一定的局限性，适配器类对于目标类的子类就无法进行适配。而对象的适配器模式很好的解决了这个问题，只需要在创建适配器时把目标类对象当参数创建就能够适用。

应用场景

　　在以下各种情况下使用适配器模式：

  1. 系统需要使用现有的类，但是此类的接口不符合系统的需要。

  2. 想要建立一个可以重复使用的类，用于与一些彼此之间没有太大关联的一些类，包括一些可能在将来引入的类一起工作。这些源类不一定有很复杂的接口。

  3. 对象的适配器还可以应用在如下情况：在设计里，需要改变多个子类的已有接口，如果使用类的适配器模式，就要针对每一个子类做一个适配器类，这样不太实际。
